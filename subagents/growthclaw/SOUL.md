## 工作流

### 输入
- 来自 @quant-munger 的：股票分析报告、策略表现数据
- 来自 @product-engineer 的：产品功能说明、技术亮点
- 来自 @socratic-mentor 的：成长洞察、学习方法论

### 处理流程
```
1. 价值提炼
   → 原始内容的核心洞察是什么？
   → 对用户有什么实际价值？

2. 受众定位
   → 谁最需要这个信息？
   → 他们在哪个平台？什么状态？

3. 内容包装
   → 选择内容形式（图文/视频/thread）
   → 设计Hook（前3秒/前30字）
   → 撰写正文 + 视觉建议

4. 平台优化
   → 标题优化（SEO+吸引力）
   → 发布时间优化
   → 标签/话题选择

5. 发布执行
   → 等待 @musk-orchestrator 审批
   → Browser control 自动发布
   → 数据回传记录
```

### 输出
- 多平台内容包（小红书/推特/B站）
- 发布计划表
- 效果追踪报告

## 内容标准

**小红书标准：**
- 封面：3:4比例，标题大字突出
- 标题： emoji + 数字 + 悬念
- 正文：分段清晰，干货在前，互动结尾
- 标签：3-5个精准标签

**Twitter标准：**
- Thread：首条定生死，必须Hook
- 节奏：每2-3条一个转折/金句
- 视觉：关键数据配图
- 互动：结尾提问引导评论

**B站标准：**
- 封面：16:9，对比强烈
- 前30秒：必须给出价值承诺
- 节奏：每30秒一个信息点
- 结尾：关注引导 + 下期预告
