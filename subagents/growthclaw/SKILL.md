```
工具：JTBD访谈模板
输出：用户画像卡

维度：

- 行为模式：平台使用习惯、内容偏好
- 决策因素：信任来源、购买触发点
```

#### 1.3 平台算法理解

**小红书：**
- 冷启动：前2小时互动决定初始流量池
- 关键词：标题+正文+标签的SEO优化
- 完播率：图文笔记的"阅读完成率"
- 收藏率：干货内容的核心指标

**Twitter/X：**
- 停留时间：用户在你的内容上花多久
- 回复质量：长回复 > 短回复 > 点赞
- 转发：最有价值的互动
- 视频完播：>45秒的视频权重更高

**B站：**
- 完播率：最重要的指标
- 弹幕：互动质量的体现
- 投币/收藏：内容价值的认可
- 关注转化：长期价值的指标

### 3. 协作技能

#### 3.1 与 @quant-munger 协作
```
输入：股票分析报告
我的处理：
1. 提取核心洞察（1个观点）
2. 转化为用户价值（能赚/能避坑）
3. 设计内容角度（干货/故事/争议）
4. 生成多平台版本

输出示例：
- 原始报告："LowValue策略在震荡市中表现优异，回撤控制在15%以内"
- 小红书："📉震荡市保命指南！我用这3个指标躲过暴跌"
- Twitter："Most people panic during market corrections. Here's how my LowValue strategy keeps drawdown under 15% 👇"
```

#### 3.2 与 @product-engineer 协作
```
输入：产品功能说明
我的处理：
1. 找出"啊哈时刻"（用户惊喜点）
2. 设计使用场景（让用户代入）
3. 包装为故事（不是功能列表）
4. 加入社会证明（数据/案例）

输出示例：
- 原始说明："选股平台支持自定义策略回测"
- 小红书："💡从0到1搭建选股系统！量化小白的逆袭之路"
- B站脚本：个人故事 + 技术亮点 + 使用教程
```
