## 核心思想一：细粒度任务设计

### 本质洞察
**把计算工作从LLM转移到确定性代码**

```
❌ 粗粒度（让LLM自己算）：
   "分析这只股票的技术面"
   → LLM需要：理解价格→计算指标→提取趋势→综合判断
   → 容易出错、推理中断

✅ 细粒度（代码算好给LLM）：
   "基于以下预计算指标做判断：
   - RoC(5d/10d/20d): [0.05, 0.03, 0.08]
   - Bollinger Z-score: 1.2
   - MACD(normalized): 0.015"
   → LLM只需：基于已知数据做金融逻辑推理
   → 更准确、可解释
```

### 关键原则
1. **禁止输入原始价格** → 必须转化为 Z-score/RoC
2. **消除价格偏见** → 所有指标除以收盘价归一化
3. **预计算 > 实时计算** → 用Pandas/NumPy算好再喂给LLM

### 应用到现有系统
- Stock Platform 当前是粗粒度（策略函数自己算指标）
- 可优化：把指标计算独立出来，标准化后作为输入
- **不需要多Agent，单Agent+细粒度输入也能提升**

## 核心思想二：分层决策架构

### 设计逻辑
模拟真实投资机构的分工：

```
Level 1: 分析师（专家分工）
   ├── 技术分析（量价数据）
   ├── 量化基本面（财务报表）
   ├── 定性分析（研报文本）
   └── 新闻情绪（事件驱动）

Level 2: 调整层（校准对齐）
   ├── 行业对标（相对优势）
   └── 宏观环境（大势判断）

Level 3: PM（综合决策）
   └── 整合所有信号，生成最终决策
```

### 关键洞察
**不是"多Agent更准"，而是"分层让每层的职责清晰"**

- 底层：专注信号提取（噪声过滤）
- 中层：专注校准对齐（语境判断）
- 顶层：专注综合决策（权重分配）

## 核心思想三：语义一致性机制

### 问题背景
**多Agent系统的"黑箱"问题**：
- Technical Agent说"动量强劲"
- PM Agent最终决策"卖出"
- 为什么？中间逻辑断裂

### 解决方案
1. **术语继承**：高层Agent必须在理由中引用底层术语
   - PM理由必须出现："动量"（来自Technical）、"盈利质量"（来自Quantitative）

2. **语义对齐监测**：
   - 计算Technical Agent与Sector Agent输出的余弦相似度
   - 阈值>0.35视为对齐（论文实证）
   - 低于阈值时降低置信度或拒绝交易

### 本质洞察
**不是"监测相似度"，而是"强制逻辑链条可追溯"**

### 应用建议
- 即使单Agent系统，也可以要求输出"分析理由"
- 建立"理由模板"，强制包含关键术语
- 人工抽查时检查逻辑链条完整性

## 核心思想四：工程实践要点

### 1. 归一化的重要性
**不仅仅是数学处理，而是"跨标的可比性"**

```python
# 普通MACD：无法比较不同价格的股票
MACD = EMA12 - EMA26  # 高价股数值大，低价股数值小

# 归一化MACD：消除价格影响
MACD_normalized = (EMA12 - EMA26) / Price  # 所有股票在同一尺度
```

**启示**：任何跨标的策略都必须考虑归一化

### 2. Temperature=1 与聚合
**不是为了"多样性"，而是为了"覆盖盲区"**

- Temperature=1：LLM输出更随机，可能捕获不同角度
- 50次试验取中位数：过滤极端值，获得稳定估计
- **但成本爆炸**：$64k/月不可持续

**替代方案**：
- Temperature=0.7 + 3次投票（成本降低90%）
- 或使用更小模型（Claude 3.5 Haiku/DeepSeek）

### 3. 数据隔离（防未来函数）
**最容易出错的地方**

论文要求：
- LLM知识截止2023年8月
- 回测起始2023年9月（严格分离）
- Agent只能访问T日及之前的数据

**启示**：A股回测必须建立严格的数据时间戳检查机制

## 关键思想 vs 工程包装

### 什么是真创新？
| 方面 | 真创新 | 工程包装 |
|------|--------|----------|
| **细粒度任务** | ✅ 把计算从LLM移到代码 | ❌ "细粒度"这个术语本身 |
| **归一化指标** | ✅ 跨标的可比性 | ❌ 复杂的数学公式 |
| **分层架构** | ✅ 职责清晰的模块化 | ❌ "多Agent"这个概念 |
| **语义一致性** | ✅ 逻辑链条可追溯 | ❌ 余弦相似度监测 |

### Quant-Munger的质疑（有道理）
> "如果做单盲测试，对照组（单Agent+预计算指标）的表现可能不会差太多"

**结论**：
- 核心提升来自**数据预处理**，而非"多Agent"
- 多Agent的复杂性可能是一种"聪明的包装"
- **可以先做单Agent+细粒度，验证有效后再考虑多Agent**
